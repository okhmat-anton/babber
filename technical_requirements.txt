# Technical Requirements: AI Agents System

## 1. Project Overview
**Concept:** A comprehensive system for building, managing, and executing AI agents. A defining feature of this project is the ability to build and configure agents using other AI agents (e.g., via VSCode extensions like Claude Opus or Gemini).
**Core Components:** Backend API, Frontend Dashboard, Agent Execution Engine, Memory System, and Skill Management.

## 2. Technology Stack
- **Backend:** Python 3.11+ with FastAPI (Async) for a high-performance, scalable API.
- **Frontend:** Vue 3 and Vuetify 3 for a responsive, user-friendly interface.
- **Database Layer:** Configurable via standard interfaces to allow flexibility:
  - *Relational/Primary:* PostgreSQL
  - *Caching/Message Broker:* Redis
  - *Vector Storage (Memory/Embeddings):* Qdrant or ChromaDB
- **File Storage:** S3-compatible storage system. Can be hosted internally (e.g., MinIO) or connected to external providers (AWS S3, Google Cloud Storage, etc.).
- **Infrastructure:** Docker and Docker Compose for containerization and deployment.
- **API Documentation:** Swagger UI / OpenAPI.
- **AI Integration:** Standardized interface for LLM providers (OpenAI, Anthropic, Google Gemini, local models, etc.), allowing seamless switching based on user needs.

## 3. Architecture & Development Standards
- **Modularity:** Standardized interfaces for databases, AI providers, and agent skills to allow seamless swapping and extension.
- **Code Quality:** PEP 8 compliance, strict typing, and linting (e.g., Ruff, Black).
- **Testing:** Comprehensive unit and integration tests to ensure reliability.
- **CI/CD:** Automated testing, linting, and Docker image building on pull requests and merges.
- **Security:** Secure handling of API keys, sandboxed execution for agent-generated code, and protection against prompt injection.
- **Performance:** Asynchronous processing for I/O-bound tasks (LLM calls, DB queries) to handle increased traffic without degradation.

## 5. Core System Features

### 5.1. Agent Management & Dashboard (Frontend)
- **Creation:** Users can create agents via the Frontend UI or by using VSCode AI assistants.
- **Monitoring:** Real-time dashboard showing agent status, active tasks, performance metrics, and progress.
- **Task Management:** Assign, pause, resume, or cancel tasks.
- **State Inspection:** View and manually edit an agent's current memory, assigned skills, and configuration.
- **Logging:** Comprehensive system and agent-specific logs (execution steps, skill usage, errors). Logs must be accessible via the Frontend and readable by VSCode agents for debugging and performance optimization.

### 5.2. Skill System
- **Definition:** Skills are modular Python scripts/functions stored in the `skills/` directory. They define specific capabilities (inputs, outputs, execution logic).
- **Standardization:** Every skill must follow a strict template/interface for easy integration, reusability, and version control.
- **Self-Education (`self_education`):** Agents can autonomously research and acquire new knowledge to improve their performance over time.
- **Skill Creation (`create_skill`):** Agents can write new Python skills based on experience.
  - Stored in the `skills/self_made/` folder.
  - Must include auto-generated documentation and usage instructions.
  - **Security:** Code generated by agents must be sandboxed, thoroughly validated, and explicitly approved before execution to prevent malicious actions.
- **Marketplace (`download_skill`):** Agents or users can download, rate, and review skills from a centralized repository/marketplace.

### 5.3. Scheduling & Automation
- **Timer Execution:** Agents can be scheduled to run specific tasks at predetermined intervals or exact timestamps (cron-like functionality).
- **Management:** User-friendly interface to view, modify, and cancel scheduled recurring tasks or one-time events.

### 5.4. Memory System
- **Architecture:** Dual-layer memory system (Short-term and Long-term) with strict privacy and security access controls.
- **Short-term Memory:** Context window management for active tasks and current conversations.
- **Long-term Memory:** Persistent storage of facts, summaries, and past experiences.
  - **Storage Logic:** Uses a hierarchical tree structure and tagging system for efficient categorization.
  - **Retrieval:** Semantic search (via Vector DB) and keyword/tag filtering to provide accurate context to the agent.
- **Learning Skills:**
  - `study <topic>`: Agent autonomously researches a topic, evaluates source credibility, summarizes key points, and stores them in long-term memory.
  - `study_from_url <url>`: Agent scrapes, parses, and summarizes content from a specific web page (handling text and metadata).
- **Summarization Engine:** Before storing, complex information is distilled into concise summaries and main points to optimize token usage and retrieval speed. Users can manually review and edit these summaries.

### 5.5. File Storage & Management
- **Storage Options:** The system supports both internal S3-compatible storage (e.g., MinIO deployed via Docker) and external cloud storage providers (AWS S3, Google Cloud Storage, etc.) for saving files, documents, and agent artifacts.
- **Configuration & Setup:**
  - **Human Admin:** Users can configure storage settings directly from the Frontend admin panel, which includes detailed, step-by-step instructions and tooltips on how to set up and connect the storage.
  - **VSCode Agent:** The storage configuration can also be fully managed and set up autonomously by the VSCode AI agent, allowing for seamless environment bootstrapping and automated infrastructure adjustments.
