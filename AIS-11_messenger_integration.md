# AIS-11: Интеграция агента с мессенджерами

**Задача:** Дать агенту возможность работать в мессенджерах и соцсетях как реальный пользователь.

**Приоритет:** Telegram (через Telethon)

---

## Обзор

Агент должен уметь работать в мессенджерах и социальных сетях как реальный пользователь. Для этого нужна система управления аккаунтами в различных платформах.

---

## Архитектура решения

### Новая вкладка "Социальные сети и мессенджеры"

Отдельная вкладка в интерфейсе агента где можно:
- Просматривать список подключенных аккаунтов
- Добавлять новые аккаунты
- Настраивать каждый аккаунт
- Смотреть статистику активности

### Структура аккаунта

Каждый подключенный аккаунт должен иметь:
- **Платформа**: Telegram, WhatsApp, Discord, VK и т.д.
- **Credentials**: API ключи, токены, session файлы
- **Статус**: активен/неактивен
- **Режимы работы**:
  - Полностью автономный
  - Только по триггерам
  - Только для доверенных пользователей

### Права доступа

#### Доверенные пользователи (whitelist)
- Могут отдавать команды агенту
- Полный доступ к skills и protocols
- Видят структурированные ответы

#### Обычные пользователи
- Агент имитирует обычного человека
- Имеет ограниченный набор разрешений (permissions)
- Могут выполнять только публичные функции:
  - Сгенерировать изображение
  - Сделать исследование/ресёрч
  - Ответить на общие вопросы
  - И т.д. (настраивается)

## Фаза 1: Telegram (Приоритет)

### Технология

**Telethon** - Python библиотека для работы с Telegram Client API

### Настройки Telegram аккаунта

```yaml
platform: telegram
api_id: "12345678"
api_hash: "abcdef1234567890"
phone: "+79001234567"
session_file: "agent_session.session"

trusted_users:
  - "@boss"
  - "@admin"
  - user_id: 123456789

public_permissions:
  - generate_image
  - web_search
  - answer_questions
  
autonomous_mode:
  enabled: true
  respond_to_mentions: true
  respond_in_groups: false
  max_daily_messages: 100

behavior:
  response_delay: [2, 8]  # секунды (имитация набора)
  typing_indicator: true
  humanize_responses: true
  casual_tone: true
```

### Функциональность

1. **Подключение**: добавление Telegram аккаунта через QR-код или код из SMS
2. **Мониторинг**: агент слушает входящие сообщения в реальном времени
3. **Триггеры**:
   - Личные сообщения
   - Упоминания в группах (@username)
   - Команды от доверенных пользователей
4. **Имитация человека**:
   - Задержка перед ответом (2-8 сек)
   - Индикатор набора текста
   - Естественный язык
   - Иногда использует эмодзи
5. **Безопасность**:
   - Игнорирование спама
   - Ограничение частоты ответов
   - Логирование всех взаимодействий

### Backend API

```
POST   /api/agents/{agent_id}/messengers
GET    /api/agents/{agent_id}/messengers
GET    /api/agents/{agent_id}/messengers/{messenger_id}
PATCH  /api/agents/{agent_id}/messengers/{messenger_id}
DELETE /api/agents/{agent_id}/messengers/{messenger_id}

# Статистика
GET    /api/agents/{agent_id}/messengers/{messenger_id}/stats
GET    /api/agents/{agent_id}/messengers/{messenger_id}/messages
```

### База данных

#### Таблица `agent_messengers`
- `id` (UUID) - первичный ключ
- `agent_id` (UUID, FK) - связь с агентом
- `platform` (string) - telegram/whatsapp/discord/etc
- `credentials` (JSON, encrypted) - API ключи, токены
- `trusted_users` (JSON array) - whitelist пользователей
- `public_permissions` (JSON array) - разрешения для обычных юзеров
- `config` (JSON) - настройки поведения
- `is_active` (boolean) - активен ли аккаунт
- `last_active_at` (timestamp) - последняя активность
- `stats` (JSON) - статистика использования

#### Таблица `messenger_messages`
- `id` (UUID) - первичный ключ
- `messenger_id` (UUID, FK) - связь с messenger аккаунтом
- `platform_message_id` (string) - ID сообщения в платформе
- `chat_id` (string) - ID чата
- `user_id` (string) - ID пользователя
- `username` (string) - username пользователя
- `direction` (enum) - incoming/outgoing
- `content` (text) - текст сообщения
- `is_command` (boolean) - это команда от доверенного?
- `response_id` (UUID, nullable) - связь с ответом
- `created_at` (timestamp) - время создания

## Технические детали

### Фоновый процесс

Для Telegram нужен постоянно работающий фоновый процесс (не HTTP запросы):

- Отдельный сервис `telegram_listener.py`
- Регистрируется как systemd/supervisor service
- Подключается к базе данных
- Для каждого активного Telegram аккаунта запускает Telethon client
- При получении сообщения вызывает агента через внутренний API

### Безопасность

- Credentials шифруются в базе (Fernet/AES)
- Session файлы хранятся в защищенной директории (`data/messengers/sessions/`)
- Rate limiting на уровне агента
- Полный аудит всех действий

### Интеграция с агентом

Когда агент получает сообщение из Telegram:

1. **Проверка whitelist** - доверенный пользователь?
2. **Если доверенный** - обрабатывает как команды (skills, protocols)
3. **Если обычный** - проверяет permissions, отвечает в casual режиме
4. **Формирование ответа** с учетом контекста и истории
5. **Отправка с имитацией человека** (задержки, typing indicator)
6. **Логирование** взаимодействия в БД
